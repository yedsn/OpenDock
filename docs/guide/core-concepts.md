# 核心概念

OpenDock 使用四级结构组织所有软件相关资源：工作空间 → 场景 → 集合 → 集合项。

## 工作空间

工作空间是 OpenDock 的顶层使用环境，用来承载用户的全部集合、资源、工具配置和插件配置。

当前版本默认只有一个本地工作空间，后续会支持多工作空间和 WebDAV 同步。

## 场景

场景是对集合的上层归类，可以理解为"某类工作上下文"。项目只是场景的一种。

常见场景示例：

| 场景 | 说明 |
|------|------|
| 官网项目 | 与官网开发相关的所有资源 |
| CAD 出图工作 | CAD 软件、图纸、输出目录 |
| 财务报表处理 | Excel 文件、报表模板、输出目录 |
| 前端开发环境 | 终端、编辑器、API 文档、本地服务 |
| 常用工具 | 日常使用的应用、网页和命令 |

场景字段：

- 场景名称
- 场景类型：项目 / 办公 / 设计 / 工程 / 通用 / 自定义
- 场景描述（可选）
- 图标/颜色（可选）

## 集合

集合是 OpenDock 最核心的管理单位，类似一个可打开的资源收藏夹。

集合可以收纳同类型资源，也可以收纳混合资源。常用集合类型：

| 集合类型 | 示例内容 | 典型打开方式 |
|----------|----------|-------------|
| 目录集合 | 项目根目录、输出目录 | 在文件管理器中打开 |
| 网页集合 | API 文档、内部工具 | 在浏览器中打开 |
| 命令集合 | 启动服务、构建脚本 | 在终端中执行 |
| 文件集合 | Office 文件、CAD 图纸 | 用关联应用打开 |
| 应用集合 | 编辑器、设计工具、数据库客户端 | 启动应用程序 |

## 集合项

集合项是集合中的单个资源条目，支持多种内置资源类型：

### 目录

本地文件夹路径。打开时在系统文件管理器中显示。

- 值：文件夹绝对路径
- 打开方式：系统默认打开

### 网页

URL 地址。打开时在浏览器中显示。

- 值：完整 URL
- 打开方式：系统默认浏览器

### 命令

可执行的 Shell 命令。打开时在终端中执行。

- 值：命令文本
- 打开方式：终端执行
- 可选：工作目录

### 文件

本地文件路径。打开时用系统关联应用打开。

- 值：文件绝对路径
- 打开方式：系统默认打开

### 应用

本地应用程序。打开时启动对应应用。

- 值：应用路径或标识
- 打开方式：启动应用程序

## 插件贡献的类型

插件可以贡献更多资源类型，例如：

- Office 文件
- CAD 图纸
- 设计文件
- 自定义业务类型

详情见：[`/guide/plugin-development`](/guide/plugin-development)

## 场景视图

场景视图按场景分组展示所有集合，帮助你快速定位目标资源。

## 快捷视图

快捷视图展示标记为快捷的集合项，适合放置最常用的资源。

## 标签和搜索

- 集合和集合项都可以添加标签
- 搜索支持中文和拼音，使用 pinyin-pro 实现
- 全局快捷键 `Alt+O` 可快速唤起搜索
